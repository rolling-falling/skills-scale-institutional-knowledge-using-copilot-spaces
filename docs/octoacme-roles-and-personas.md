# OctoAcme Personas

This document defines typical roles and responsibilities used in OctoAcme project docs and exercises.

---

## Developers

### Role Summary
Developers design, build, test, and deliver software components. They collaborate with product and project leads to implement features that meet acceptance criteria and quality standards.

### Responsibilities
- Implement features and fixes to meet acceptance criteria
- Write and maintain tests and documentation
- Participate in design and code reviews
- Assist in estimating and planning work
- Help identify technical risks and propose mitigations

### Goals
- Deliver reliable, maintainable code
- Reduce cycle time from idea to production
- Maintain high test coverage and observability

### Typical Communication
- Daily standups and sprint planning
- PR descriptions and code review comments
- Technical design docs when needed

---

## Product Managers

### Role Summary
Product Managers define what should be built to deliver customer and business value. They own the product vision, prioritize the backlog, and measure outcomes.

### Responsibilities
- Define problem statements and success metrics
- Prioritize the roadmap and backlog
- Collaborate with stakeholders and engineering on trade-offs
- Validate solutions through user research and metrics

### Goals
- Maximize customer value and impact
- Make clear, data-driven prioritization decisions
- Ensure product-market fit and usability

### Typical Communication
- Weekly alignment with PM and engineering leads
- Roadmap updates and stakeholder briefings
- Acceptance criteria and feature specs

---

## Project Managers

### Role Summary
Project Managers coordinate delivery activities, manage schedules, risks, and communications. They enable the team to deliver on commitments efficiently.

### Responsibilities
- Create and maintain project plans and timelines
- Manage risks, dependencies, and resource constraints
- Facilitate meetings (kickoff, planning, retrospectives)
- Ensure consistent project documentation and status reporting
- Coordinate cross-team and stakeholder communication

### Goals
- Deliver projects on time and within scope
- Minimize unplanned work and escalations
- Maintain transparency and alignment across stakeholders

### Typical Communication
- Weekly status updates and stakeholder reports
- Risk registers and decision logs
- Coordination via project boards and meeting facilitation

---

## UX Designer

### Role Summary
UX Designers ensure user features are intuitive and support a positive user experience by collaborating with product and development teams.

### Responsibilities
- Design flows, wireframes, and prototypes
- Apply user feedback to the design process
- Ensure accessibility standards are met
- Conduct usability testing and iterate on designs
- Maintain design system consistency

### Goals
- Create intuitive and accessible user experiences
- Reduce user friction and improve satisfaction
- Ensure design consistency across features

### Typical Communication
- Design review sessions with developers and product managers
- User research findings and usability test results
- Design specs and prototype walkthroughs
- Accessibility audit reports

### Interactions
Works with Product Managers and Developers throughout planning and review stages, collaborates with QA Engineers on usability testing, and coordinates with Technical Writers on user-facing documentation.

---

## Technical Writer

### Role Summary
Technical Writers create and update accurate documentation for internal and user-facing artifacts.

### Responsibilities
- Write technical and process documentation
- Collaborate with developers and product managers on accuracy
- Review documentation for releases
- Maintain documentation standards and style guides
- Update help content and API documentation

### Goals
- Ensure documentation is accurate, clear, and up-to-date
- Support developer onboarding and productivity
- Improve user self-service and support efficiency

### Typical Communication
- Documentation reviews with developers
- Release notes and changelog updates
- User guide and tutorial content
- Documentation feedback from customer support

### Interactions
Coordinates with Developers, Product Managers, and Customer Support Liaison. Works with Release Manager during release cycles and UX Designer for user-facing content alignment.

---

## QA Engineer

### Role Summary
QA Engineers own the definition and verification of quality for features and releases.

### Responsibilities
- Develop test cases and test plans
- Execute manual and automated tests
- Report and track defects
- Perform regression testing
- Validate acceptance criteria compliance

### Goals
- Ensure features meet quality standards before release
- Catch defects early in the development cycle
- Maintain comprehensive test coverage

### Typical Communication
- Bug reports and defect tracking
- Test plan reviews and coverage reports
- Quality gate approvals
- Test automation strategy discussions

### Interactions
Works closely with Developers during build and release phases, collaborates with Product Managers on acceptance criteria validation, coordinates with Release Manager for deployment quality gates, and partners with UX Designer on usability testing.

---

## Release Manager

### Role Summary
Release Managers manage the release process, ensuring smooth deployments and clear communications.

### Responsibilities
- Schedule and coordinate release activities
- Oversee release checklists and rollback plans
- Announce releases to stakeholders
- Manage release branches and version control
- Coordinate deployment windows and maintenance schedules

### Goals
- Ensure reliable and predictable releases
- Minimize deployment risks and downtime
- Maintain clear release communication

### Typical Communication
- Release schedules and deployment plans
- Go/no-go decision meetings
- Release retrospectives and incident reviews
- Stakeholder release announcements

### Interactions
Collaborates with QA Engineers, Developers, and Product Managers for production releases. Coordinates with Customer Support Liaison for release communications and Technical Writer for release notes.

---

## Customer Support Liaison

### Role Summary
Customer Support Liaisons bring user insights and support feedback into planning and improvement cycles.

### Responsibilities
- Communicate user-reported issues to the team
- Provide feedback on releases from support perspective
- Review documentation for user clarity
- Escalate critical customer issues
- Track common support requests and trends

### Goals
- Ensure user voice is represented in product decisions
- Reduce support ticket volume through product improvements
- Improve user satisfaction and support efficiency

### Typical Communication
- Weekly support trends and issue summaries
- Customer feedback sessions
- Documentation and help content reviews
- Critical issue escalations

### Interactions
Links the Support team with Product Managers and engineering. Works with Technical Writers on documentation clarity, UX Designers on usability issues, and Release Manager on release communications.

---

## Security Champion

### Role Summary
Security Champions advocate for secure practices, surface risks, and help manage incidents.

### Responsibilities
- Review new features for security risks
- Promote secure development practices
- Respond to security incidents
- Conduct security code reviews
- Maintain security documentation and runbooks

### Goals
- Prevent security vulnerabilities in production
- Build security awareness across the team
- Ensure rapid response to security incidents

### Typical Communication
- Security review feedback on PRs and designs
- Security incident reports and postmortems
- Secure coding guidelines and training
- Threat model discussions

### Interactions
Advises Developers and Product Managers on security best practices, coordinates with security team on incident response, works with Project Managers on security-related risks, and collaborates with QA Engineers on security testing.

---

## Data Analyst

### Role Summary
Data Analysts enable data-driven project decisions through analysis and reporting.

### Responsibilities
- Analyze usage and success metrics
- Identify trends and suggest improvements
- Generate periodic reports and dashboards
- Define and track KPIs
- Support A/B testing and experiments

### Goals
- Provide actionable insights for product decisions
- Measure and report on success metrics
- Identify opportunities for optimization

### Typical Communication
- Weekly/monthly metrics reports
- A/B test results and analysis
- Dashboard reviews and metric definitions
- Data insights presentations

### Interactions
Supports Product Managers, Developers, and Leadership with actionable data. Works with QA Engineers on quality metrics and Customer Support Liaison on user behavior trends.
