# OctoAcme Role Onboarding Checklist

## Purpose
This document provides role-specific onboarding checklists to help new team members quickly understand their responsibilities, key touchpoints, and essential processes for OctoAcme projects.

Related to issue #5: https://github.com/rolling-falling/skills-scale-institutional-knowledge-using-copilot-spaces/issues/5

---

## General Onboarding (All Roles)

**Completed by**: Day 1-3

- [ ] **Read core documentation:**
  - [ ] [Project Management Overview](octoacme-project-management-overview.md)
  - [ ] [Roles and Personas](octoacme-roles-and-personas.md) (focus on your role)
  - [ ] Current project's One-pager and README
- [ ] **Access setup:**
  - [ ] Repository access granted
  - [ ] Project board/tracking tool access
  - [ ] Communication channels (Slack, email groups, etc.)
  - [ ] Relevant dashboards and monitoring tools
- [ ] **Introductions:**
  - [ ] Meet with Project Manager for role overview
  - [ ] Introduction to team members and their roles
  - [ ] Understand current project status and priorities
- [ ] **Review escalation paths** in [Risks and Communication](octoacme-risks-and-communication.md)

## Project Manager Onboarding

**Accountability**: Own project execution, coordination, and communication

### Week 1
- [ ] Review all project documentation:
  - [ ] Project Charter/One-pager
  - [ ] Risk Register
  - [ ] Project timeline and milestones
  - [ ] Meeting cadence and stakeholder list
- [ ] Understand team composition and role assignments
- [ ] Shadow current PM (if transitioning) on:
  - [ ] Daily standup
  - [ ] Weekly PM/PdM sync
  - [ ] Stakeholder update
- [ ] Access all project management tools and templates

### Week 2-4
- [ ] Lead daily standup (with support if transitioning)
- [ ] Conduct risk register review
- [ ] Send first weekly status update
- [ ] Meet 1:1 with each team member to understand their needs
- [ ] Review and update project timeline
- [ ] Identify and document any process gaps or improvements

### Key Touchpoints
- **Daily**: Standup facilitation
- **Weekly**: PM/PdM sync, risk review, status updates
- **As needed**: Escalations, stakeholder communications

### Essential Skills
- Coordination and facilitation
- Risk management
- Clear communication
- Conflict resolution

---

## Product Manager Onboarding

**Accountability**: Own product vision, backlog prioritization, and success metrics

### Week 1
- [ ] Review product roadmap and vision
- [ ] Understand success metrics and how they're tracked
- [ ] Review current backlog and prioritization rationale
- [ ] Meet with Data Analyst to understand metrics/analytics
- [ ] Understand customer feedback channels and insights

### Week 2-4
- [ ] Shadow stakeholder meetings and customer research sessions
- [ ] Review and refine backlog with team input
- [ ] Conduct acceptance criteria review for in-flight work
- [ ] Meet with UX Designer to understand design vision
- [ ] Meet with Customer Support Liaison to understand user pain points

### Key Touchpoints
- **Weekly**: PM/PdM sync, backlog grooming
- **Sprint/iteration**: Planning sessions, demos
- **Monthly**: Metrics review with Data Analyst, stakeholder briefings

### Essential Skills
- Prioritization and trade-off decisions
- User empathy and research
- Data-driven decision making
- Stakeholder management

---

## Developer Onboarding

**Accountability**: Deliver quality code that meets acceptance criteria

### Week 1
- [ ] Set up development environment
- [ ] Review codebase architecture and conventions
- [ ] Read [Execution and Tracking](octoacme-execution-and-tracking.md) for PR and workflow guidance
- [ ] Understand branching strategy and CI/CD pipeline
- [ ] Review Definition of Done
- [ ] Meet with Tech Lead for architecture overview

### Week 2-4
- [ ] Complete first small task/bug fix
- [ ] Submit first PR following team conventions
- [ ] Participate in code reviews
- [ ] Pair with experienced developer on medium-sized feature
- [ ] Understand testing requirements (unit, integration, E2E)
- [ ] Learn security scanning process with Security Champion

### Key Touchpoints
- **Daily**: Standup participation
- **As needed**: Code reviews, pairing sessions, design discussions
- **Sprint**: Planning, demos

### Essential Skills
- Code quality and testing
- Collaboration and code review
- Estimation and planning
- Technical design

---

## UX Designer Onboarding

**Accountability**: Ensure intuitive and accessible user experience

### Week 1
- [ ] Review design system and style guides
- [ ] Understand user personas and user research findings
- [ ] Review current designs and in-flight design work
- [ ] Access design tools (Figma, Sketch, etc.)
- [ ] Meet with Product Manager to understand product vision
- [ ] Review accessibility standards and requirements

### Week 2-4
- [ ] Shadow user research or usability testing session
- [ ] Review design with Developers to understand implementation constraints
- [ ] Create first design or iterate on existing work
- [ ] Participate in design review session
- [ ] Work with QA Engineer on usability testing approach

### Key Touchpoints
- **Planning**: Design review sessions
- **Weekly/bi-weekly**: Design critiques, user research updates
- **As needed**: Developer consultations, accessibility reviews

### Essential Skills
- User-centered design
- Wireframing and prototyping
- Accessibility standards
- Collaboration with engineering

---

## Technical Writer Onboarding

**Accountability**: Maintain accurate, clear, and up-to-date documentation

### Week 1
- [ ] Review existing documentation (user guides, API docs, process docs)
- [ ] Understand documentation standards and style guide
- [ ] Access documentation tools and repositories
- [ ] Meet with Developers to understand technical architecture
- [ ] Review upcoming features requiring documentation
- [ ] Understand release schedule and documentation deadlines

### Week 2-4
- [ ] Update or create first piece of documentation
- [ ] Get feedback from Developers and Product Manager
- [ ] Participate in design/feature review to understand documentation needs
- [ ] Shadow Customer Support Liaison to understand common user questions
- [ ] Review documentation with UX Designer for user-facing content alignment

### Key Touchpoints
- **Pre-release**: Documentation reviews with developers
- **Release**: Release notes preparation with Release Manager
- **As needed**: Feature documentation, process updates

### Essential Skills
- Technical writing and clarity
- Audience understanding
- Collaboration with engineers
- Documentation tools

---

## QA Engineer Onboarding

**Accountability**: Ensure features meet quality standards and acceptance criteria

### Week 1
- [ ] Review test plans and testing approach
- [ ] Understand test automation framework and tools
- [ ] Access test environments (staging, QA, etc.)
- [ ] Review Definition of Done and quality gates
- [ ] Meet with Developers to understand codebase and test coverage
- [ ] Understand defect tracking process

### Week 2-4
- [ ] Execute manual tests for current features
- [ ] Review and run automated test suite
- [ ] Report first defects following team process
- [ ] Participate in acceptance criteria review during planning
- [ ] Work with UX Designer on usability testing
- [ ] Shadow release process with Release Manager

### Key Touchpoints
- **Daily**: Standup, testing progress updates
- **Planning**: Acceptance criteria review, test planning
- **Pre-release**: QA sign-off, smoke testing
- **Demo**: Feature validation and feedback

### Essential Skills
- Test planning and execution
- Defect reporting and tracking
- Automation (where applicable)
- Attention to detail

---

## Release Manager Onboarding

**Accountability**: Ensure reliable and smooth releases to production

### Week 1
- [ ] Review [Release and Deployment Guide](octoacme-release-and-deployment.md)
- [ ] Understand release process and deployment pipeline
- [ ] Access deployment tools and production systems
- [ ] Review release schedule and upcoming releases
- [ ] Meet with QA Engineer to understand quality gates
- [ ] Understand rollback procedures and incident response

### Week 2-4
- [ ] Shadow a release deployment (staging and production)
- [ ] Review release checklist and rollback plan
- [ ] Coordinate with Customer Support Liaison on release communication
- [ ] Work with Technical Writer on release notes
- [ ] Conduct post-release retrospective
- [ ] Meet with Security Champion on security review requirements

### Key Touchpoints
- **Weekly**: Release planning, deployment coordination
- **Pre-release**: Go/no-go meetings, deployment preparation
- **Release day**: Deployment execution, monitoring
- **Post-release**: Retrospective, communication

### Essential Skills
- Deployment and operations
- Risk management
- Coordination and communication
- Incident response

---

## Customer Support Liaison Onboarding

**Accountability**: Bridge support team and product team with user insights

### Week 1
- [ ] Understand support ticket system and common user issues
- [ ] Review support escalation process
- [ ] Meet with Product Manager to understand product priorities
- [ ] Access support documentation and knowledge base
- [ ] Understand how user feedback is collected and shared
- [ ] Review upcoming releases and user-impacting changes

### Week 2-4
- [ ] Attend support team meetings to understand user trends
- [ ] Share first user insights with product team
- [ ] Review documentation with Technical Writer for clarity
- [ ] Participate in demo to provide support perspective
- [ ] Shadow release communication process
- [ ] Work with UX Designer on usability issues from support tickets

### Key Touchpoints
- **Weekly**: Support trends report to Product Manager
- **Pre-release**: Support team briefing, documentation review
- **As needed**: Critical issue escalation, user feedback sessions

### Essential Skills
- User empathy and support
- Communication and translation
- Issue triage and escalation
- Documentation review

---

## Security Champion Onboarding

**Accountability**: Advocate for secure practices and manage security risks

### Week 1
- [ ] Review security policies and secure development guidelines
- [ ] Understand security scanning tools in CI/CD
- [ ] Access security dashboards and vulnerability tracking
- [ ] Meet with Tech Lead to understand architecture and security touchpoints
- [ ] Review security incident runbook and escalation path
- [ ] Understand threat model for current project

### Week 2-4
- [ ] Review code for security issues (PR reviews)
- [ ] Conduct first security review of a feature or design
- [ ] Participate in design review with security lens
- [ ] Work with Developers on remediating security findings
- [ ] Shadow security incident response (if one occurs)
- [ ] Conduct security awareness session with team

### Key Touchpoints
- **As needed**: Security reviews on PRs and designs
- **Weekly**: Security scan review, risk register updates
- **Incidents**: Immediate response and coordination

### Essential Skills
- Security knowledge and best practices
- Risk assessment
- Secure code review
- Incident response

---

## Data Analyst Onboarding

**Accountability**: Enable data-driven decisions through analysis and reporting

### Week 1
- [ ] Understand data sources and analytics tools
- [ ] Review existing dashboards and metrics
- [ ] Meet with Product Manager to understand success metrics
- [ ] Access data warehouse, BI tools, and analytics platforms
- [ ] Review data privacy and compliance requirements
- [ ] Understand current reporting cadence

### Week 2-4
- [ ] Create or update first dashboard or report
- [ ] Conduct data analysis to answer product question
- [ ] Present insights to Product Manager or team
- [ ] Work with Developers on instrumentation/tracking
- [ ] Review A/B test or experiment results (if applicable)
- [ ] Identify data quality issues or measurement gaps

### Key Touchpoints
- **Weekly/Monthly**: Metrics reviews with Product Manager
- **Planning**: Help define success metrics and KPIs
- **As needed**: Ad-hoc analysis, experiment results

### Essential Skills
- Data analysis and visualization
- Statistical thinking
- Communication of insights
- Tool proficiency (SQL, BI tools, etc.)

## Cross-Role Collaboration Guide

Understanding how your role interacts with others is critical for effective collaboration:

| Your Role | Primary Collaborators | Key Interactions |
|-----------|----------------------|------------------|
| **Project Manager** | All roles | Coordination, status updates, escalations |
| **Product Manager** | Project Manager, UX Designer, Data Analyst, Customer Support Liaison | Vision, prioritization, metrics |
| **Developer** | Tech Lead, QA Engineer, UX Designer, Security Champion | Code delivery, reviews, design implementation |
| **UX Designer** | Product Manager, Developers, QA Engineer, Technical Writer | Design reviews, usability, user research |
| **Technical Writer** | Developers, Product Manager, Customer Support Liaison | Documentation reviews, release notes |
| **QA Engineer** | Developers, Release Manager, UX Designer | Testing, quality gates, acceptance validation |
| **Release Manager** | QA Engineer, Developers, Technical Writer, Customer Support Liaison | Deployment coordination, release communication |
| **Customer Support Liaison** | Product Manager, Technical Writer, UX Designer | User insights, documentation review |
| **Security Champion** | Developers, Project Manager | Security reviews, incident response |
| **Data Analyst** | Product Manager, Developers | Metrics definition, analysis, reporting |
